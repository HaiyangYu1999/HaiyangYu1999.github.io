设计一个类, 我们只能生成该类的一个实例

1. 饿汉式

能防住多线程, 不能防住反射.

2. 懒汉式

不能防住多线程, 不能防住反射.

3. 加锁懒汉式

4. 加锁加volatile懒汉式

多线程足够安全, 仍然防不住反射

5. 静态内部类实现单例模式

就是把饿汉式的new 单例模式的过程private final static Hungry HUNGRY = new Hungry();放到个静态内部类中, 通过静态内部类的加载机制实现懒惰加载. 仍然防不住反射

6. 枚举类实现单例模式

这个最安全, 多线程和反射都破坏不了, 反序列化的方法也破解不了. 也最简洁, 核心代码只需要3行. 加载模式和饿汉式一样, 都是提前加载. 怪不得都推荐使用这种方法
